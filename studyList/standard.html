NSSOL前端项目编程规范

（本规范节选并翻译了NS Solution社内公约中被认为“必须遵守”的部分）
详细请参阅：《クライアントサイド開発コーディング規約(HTML_CSS_JavaScript) 》

目的

1、开发系统，书写代码，最低的要求是完成功能的实现。 但最好的代码应该是逻辑清晰，并包含适当的注释。 不仅自己能看得懂，其它程序员也能很容易明白。

2、可维护的代码 可维护的代码使得后续的开发和代码的重用性变得简单，在 整体的项目开发中达到成本的节约。 前端开发也是如此，因此制定一系列的规范。

基础规范

html、CSS、JavaScript的文件分离
html：视图结构内容
CSS： 视图样式显示
JavaScript：用户交互动作实现

hifive框架规范

MVC：将数据、视图、控制器的代码分离
Controller：控制交互和视图的改变，不能计算等逻辑处理。
Logic：负责给Controller提供逻辑计算、数据加载等方法。
View：负责视图的内容（HTML或者单独的ejs模板）
Model：负责数据的管理（数据的属性类型等）

代码规范

  • 命名规范:变量常量函数的命名要有意义，语义化
    • 变量：
      UserID、UserPassword （驼峰式命名）
    • 常量：
      MAX_COUNT（全部大写，单词用下划线分割）
    • 函数：
      getUserID（）、setPassword（）、canRead（）
  • 注释规范:单行注释（//）
    • 单独一行： //(双斜线)与注释文字之间保留一个空格
      例：
// 设置标题
setTitle();
    • 在代码后面添加注释：//(双斜线)与代码之间保留一个空格，并且//(双斜线)与注释文字之间保留一个空格。
      例：
var maxCount = 10; // 设置最大量
    • 注释代码：//(双斜线)与代码之间保留一个空格。
      例：
// setName(); // 注释代码
  • 注释规范:多行注释 ( / 注释说明 / )
    • 若开始(/*和结束(*/)都在一行，推荐采用单行注释
    • 若至少三行注释时，第一行为/*，最后行为*/，其他行以*开始，并且注释文字与*保留一个空格。 
      例
/*
    * 代码执行到这里后会调用setTitle()函数
    * setTitle()：设置title的值
    */
    • 函数 ( 方法 ) 注释 
      例
/** 
     * 函数说明 
     * @关键字 
     */
  • HTML规范
    • 文档声明： 使用 HTML5 的文档声明类型 :
    • 语义化：
      语义化是指：根据元素其被创造出来时的初始意义来使用它。
      意思就是用正确的标签干正确的事，而不是只有div和span
  • JavaScript规范
    • 避免全局命名空间污染：
      防止全局命名空间被污染，我们通常的做法是将代码包裹成一个 IIFE(Immediately-Invoked Function Expression)，创建独立隔绝的定义域。也使得内存在执行完后立即释放。 
      (function(){
	…
})();
    • 声明提前:
      javascript会自动将函数作用域内的变量和方法的定义提前（只是提前声明，赋值还是在原处）
  • CSS规范:
    • id和class的命名：
      ID和class的名称总是使用可以反应元素目的和用途的名称，或其他通用的名称，代替表象和晦涩难懂的名称
    • 合理的使用ID：
      一般情况下ID不应该被用于样式，并且ID的权重很高，所以不使用ID解决样式的问题，而是使用class
    • css选择器中避免使用标签名：
      从结构、表现、行为分离的原则来看，应该尽量避免css中出现HTML标签，并且在css选择器中出现标签名会存在潜在的问题。

总结

代码规范是为了增加代码的可读性，可维护性所采取的非常重要措施。 材料中记载的只是一些最基本的规范，在实际项目中通常会有来自于企业，项目团队的更为详细和明确的规范要求。

Formatter格式的利用

当我们开发javascript前端项目的时候，代码格式非常重要，有了代码格式规范，可以使得我们的项目代码看上去紧凑、整洁。为了符合本公司的代码格式规范，当我们开发项目的时候要使用我们公司自己的代码格式。接下来介绍一下本公司的代码格式formatter。

Formatter格式规范的下载

点击这里下载H5_JS_WST_FORMATTER_PROFILE格式文件下载后请解压保存到自己新建的文件中，方便接下来的导入。

Formatter格式的导入

  • 1.首先导入一个js文件，点击左上角的File，然后在列表中选择open project点击

  • 2.随后会出现如下画面，点击directory选择项目
  • 3.右击文件，点击properties
  • 4.随后出现以下画面，请按照以下顺序执行：
    • 第一步：点击JavaScript
    • 第二步：点击Code Style
    • 第三步：点击Formatter
    • 第四步：勾选enbale project specific settings
    • 第五步：点击Import

  • 5.导入下载下来解压后的H5_JS_WST_FORMATTER_PROFILE.xml文件
  • 6.设置文件保存时自动完成该格式的设置，请按照以下顺序执行:
    • 第一步：点击JavaScript
    • 第二步：点击Editor
    • 第三步：点击Save Actions
    • 第四步：勾选Enbale project specific settings
    • 第五步：勾选Perform the selected actions on save
    • 第六步：勾选Format source code

  • 通过以上步骤，当你Ctrl+S保存文件的时候，文件会自动按照format设定的格式规范文件。
